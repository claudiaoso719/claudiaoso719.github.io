Academic Projects

Computational Jigsaw Puzzle Solver

Built a computer vision system that automatically solves jigsaw puzzles by extracting piece contours, matching edges using Mean Squared Error analysis, and assembling solutions through greedy graph traversal. Implemented rotation handling with perspective transformation and edge blur correction for rotated pieces.

Key Features: Contour detection and piece extraction, MSE-based edge matching with relaxed matching algorithm, greedy BFS assembly with rotation support, animated solution visualization

Impact: Successfully solved puzzles with both uniform translation and rotation, achieving sub-second assembly times

Technologies: Python, OpenCV, NumPy, Matplotlib

Multimedia Systems Design • Fall 2025

Network File Streaming Service

Built a network file streaming service with HTTP server and intelligent caching system for game asset loading. The demo shows side-by-side spawning of car meshes - the right car loads locally while the left car streams from the network server. Real-time performance metrics display load times, demonstrating the caching system's impact: first network load takes ~0.13 seconds, subsequent cached loads complete nearly instantly (~0.01 seconds). The demo shows cache clearing and toggling to illustrate performance differences with and without caching enabled.

Key Features: HTTP-based asset server, client-side caching with runtime toggle and cache clearing, real-time performance metrics display, visual differentiation between local and network-loaded assets

Impact: Demonstrated understanding of asset pipeline optimization and network-based content delivery for game development

Technologies: C++, Python HTTP Server, Caching Systems, Lua API Integration, PrimeEngine

Game Engine Development • Spring 2025

Hot Reloading System

Built as part of my game engine coursework, this real-time script hot reloading pipeline enables developers to see code changes instantly without application restarts. The demo shows a soldier tracking and shooting at a moving target controlled by Lua script. Mid-gameplay, I modify the script to stop the target's movement and reload it - the soldier continues tracking but the target immediately stops, demonstrating live script updates without restarting the game.

Key Features: Script Registry with file monitoring, rolling hash-based change detection, seamless runtime script execution via Lua C API

Impact: Eliminated need for application restarts during development, enabling rapid iteration on game logic and behavior

Technologies: C++, Lua C API, Rolling Hash Algorithm, PrimeEngine

Network Snapshot/Diff Tool

Built a pipeline tool for Houdini that tracks and compares changes in node networks across different workflow stages. Captures comprehensive snapshots including node parameters, connections, and positions, then performs differential analysis to identify additions, deletions, and modifications with visual highlighting in the viewport.

Key Features: Automated network state capture with parameter tracking, comparative analysis between snapshots with visual diff display, custom PySide6 UI with dropdown-based snapshot selection, color-coded viewport highlighting (green highlights added nodes, yellow highlights modified nodes)

Impact: Enables artists to track workflow iterations and debug complex node networks by providing clear visualization of changes between different setup stages

Technologies: Python, PySide6, Houdini Python API (HOM), JSON serialization, Qt Designer
